# Certification & Skills Roadmap — Cloud / DevOps

Career target: **Cloud Engineer / DevOps Engineer**, building up from an entry-level role.

This roadmap is the plan *after* AZ-900. Revisit and commit to a path once AZ-900 is passed.

## Current position

- **AZ-900 (Azure Fundamentals)** — booked for 29 May 2026. The foundation cert: proves you can name and reason about cloud concepts. First piece of outside validation on top of the homelab work.
- **Homelab** — built and documented (git, virtualisation, Linux, Docker). Already demonstrates the "show the work" instinct.

## The path after AZ-900

AZ-900 is the only pure "fundamentals" cert needed. Everything after it is **role-based** — and the two career targets split slightly.

### Next step (both targets): AZ-104 — Azure Administrator

The natural follow-on. This is where you stop *describing* Azure and start *doing* it: managing identities, storage, virtual networks, VMs, monitoring. A real step up in difficulty — hands-on and scenario-heavy. This is the cert that moves you off entry-level, because it maps to a job employers recognise. For a **Cloud Engineer** target, this is the spine.

### Then the fork

- **DevOps Engineer** → **AZ-400 (DevOps Engineer Expert)** — the core DevOps cert: CI/CD pipelines, infrastructure as code, release management. Prerequisite: AZ-104 *or* AZ-204 (Developer). Path: AZ-900 → AZ-104 → AZ-400.
- **Cloud Engineer** → AZ-104 may be enough cert-wise for a while; deepen with hands-on work rather than immediately stacking another exam.

## What matters more than the next cert

Between AZ-900 and AZ-104, the real differentiator is **skills the certs don't test**:

- **Linux** — already have this (homelab).
- **Terraform** — infrastructure as code.
- **Docker / Kubernetes** — containers and orchestration.
- **A scripting language** — e.g. Python or Bash.
- **Git / CI workflows** — version control and pipelines.

A candidate with AZ-104 *and* a Terraform-built project beats a candidate with three certs and nothing to show. The homelab repo is already this instinct working correctly.

## Realistic sequence

1. **AZ-900** — pass it (29 May 2026).
2. **Hands-on cloud project** — add something real to the repo (e.g. Terraform-provisioned Azure resources).
3. **AZ-104** — the role-based step up.
4. **Decide DevOps vs Cloud Engineer** and specialise (AZ-400 if DevOps).

Key principle: **don't exam-stack.** Alternate certs with visible projects.
